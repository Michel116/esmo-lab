
# Обзор архитектуры и кода приложения "ЭСМО Калибровка"

## 1. Введение

ЭСМО Калибровка — это веб-приложение, предназначенное для упрощения и систематизации процесса ввода данных с различных измерительных приборов. Оно позволяет пользователям выбирать тип устройства, вводить показания в адаптированные формы и сохранять данные локально в браузере. Приложение также поддерживает управление списком поверителей и экспорт собранных данных в формат Excel.

## 2. Технологический стек

Приложение построено на современном стеке веб-технологий:

Next.js (v15.x): Реакт-фреймворк для создания серверных и статических веб-приложений. Используется App Router.
React (v18.x): Библиотека JavaScript для построения пользовательских интерфейсов. Активно используются функциональные компоненты и хуки.
TypeScript: Язык программирования, добавляющий статическую типизацию в JavaScript, что улучшает читаемость и надежность кода.
ShadCN UI: Коллекция готовых, стильных и настраиваемых компонентов пользовательского интерфейса, построенных на базе Tailwind CSS и Radix UI. Компоненты хранятся локально в src/components/ui/.
Tailwind CSS: Утилитарный CSS-фреймворк для быстрой стилизации интерфейса.
Lucide React: Библиотека иконок.
LocalStorage: Все данные (записи поверок, список поверителей) хранятся локально в браузере пользователя, обеспечивая приватность и доступность данных без необходимости подключения к серверу.
date-fns: Библиотека для работы с датами (форматирование).
xlsx: Библиотека для генерации Excel-файлов (.xlsx) для экспорта данных.

## 3. Структура проекта

Основные каталоги и файлы проекта:

src/: Корневая папка с исходным кодом приложения.
    app/: Содержит основные страницы и маршрутизацию приложения (согласно Next.js App Router).
        layout.tsx: Главный макет приложения, оборачивающий все страницы. Подключает глобальные стили, шрифты и компонент Toaster для уведомлений.
        globals.css: Глобальные стили и CSS-переменные для темы оформления (HSL значения для цветов фона, текста, акцентов и т.д.).
        page.tsx: Главная (стартовая) страница приложения. Представляет собой "визитку" с описанием возможностей и кнопкой для перехода к панели управления. Реализует диалог для выбора или добавления поверителя перед переходом.
        dashboard/page.tsx: Основная рабочая страница ("Панель управления данными"). Здесь происходит выбор устройства, ввод данных, управление поверителями и просмотр таблицы с собранными данными. Большая часть логики этой страницы вынесена в клиентский компонент DashboardClientPage.
    components/: Содержит переиспользуемые React-компоненты.
        ui/: Компоненты из библиотеки ShadCN UI (кнопки, карточки, диалоги, инпуты и т.д.). Они были добавлены через CLI ShadCN и могут быть кастомизированы.
        AppLogo.tsx: Компонент для отображения логотипа приложения.
        dashboard/: Компоненты, специфичные для страницы "Панель управления данными".
            DataEntryForm.tsx: Ключевой компонент, отвечающий за отображение формы ввода данных в зависимости от выбранного типа устройства. Содержит сложную логику для пошагового ввода данных для термометров и алкотестеров, управление показаниями, расчеты и взаимодействие с пользователем (диалоги, уведомления).
            DataTable.tsx: Компонент для отображения собранных данных в виде таблицы с возможностью группировки по типу устройства, раскрытия деталей записей, удаления и экспорта в Excel.
            InspectorManager.tsx: Компонент для управления списком поверителей (добавление, удаление, выбор текущего).
        layout/: Компоненты, связанные с макетом страниц.
            Navbar.tsx: Верхняя навигационная панель приложения.
    config/: Файлы конфигурации.
        devices.ts: Центральный файл конфигурации для всех поддерживаемых типов измерительных приборов. Определяет их ID, имя, описание, иконку, а также специфические настройки, такие как поля для стандартных устройств или контрольные точки для термометров и алкостестеров (включая эталонные значения и допустимые пределы).
    hooks/: Пользовательские React-хуки.
        use-toast.ts: Хук для управления системой уведомлений (тостов). Позволяет отображать сообщения об успехе, ошибках и т.д.
        use-mobile.ts: (Не используется в текущей версии активно, но был частью UI kit sidebar) Хук для определения, отображается ли приложение на мобильном устройстве.
    lib/: Вспомогательные утилиты.
        utils.ts: Содержит функцию cn для условного объединения CSS-классов (используется Tailwind Merge и clsx).
    types/: Определения TypeScript-типов.
        index.ts: Содержит все кастомные типы, используемые в приложении (например, DataEntry, Inspector, DeviceConfig, DeviceId, типы для точек поверки термометров и алкотестеров). Это помогает обеспечить типобезопасность и лучшую читаемость кода.
public/: Папка для статических файлов (изображения, шрифты и т.д.), которые доступны напрямую по URL.
next.config.ts: Файл конфигурации Next.js (настройки сборки, игнорирование ошибок TypeScript/ESLint при сборке, конфигурация для удаленных изображений).
package.json: Определяет зависимости проекта, скрипты для запуска, сборки и линтинга.
tailwind.config.ts: Файл конфигурации Tailwind CSS (определение кастомных шрифтов, цветов из CSS-переменных, анимаций).
components.json: Файл конфигурации для ShadCN UI, указывающий стиль, пути к компонентам и настройки Tailwind.
tsconfig.json: Файл конфигурации TypeScript.

## 4. Управление данными и состоянием

### 4.1. Хранение данных (LocalStorage)

Приложение использует localStorage браузера для хранения всех пользовательских данных:
Записи поверок (datafill-entries): Массив объектов DataEntry, где каждая запись содержит информацию о серийном номере, типе прибора, введенных показаниях, поверителе и времени.
Список поверителей (datafill-inspectors): Массив объектов Inspector.
Выбранный поверитель (datafill-selectedInspectorId): ID текущего выбранного поверителя.
Выбранное устройство (datafill-selectedDevice): ID последнего выбранного типа устройства.
Импортированные S/N термометров (datafill-imported-thermo-serials): Список серийных номеров, импортированных из Excel для термометров.

Данные загружаются из localStorage при инициализации страницы DashboardClientPage и сохраняются обратно при каждом их изменении (используются хуки useEffect).

### 4.2. Управление состоянием на клиенте

Основное состояние приложения управляется в компоненте src/app/dashboard/page.tsx (точнее, в его клиентской части DashboardClientPage) с помощью хуков React:
useState: Для хранения списков поверителей, записей, выбранного поверителя, выбранного устройства, текущих значений формы, шагов в многошаговых формах (термометры, алкотестеры) и т.д.
useEffect:
    Для загрузки и сохранения данных в localStorage.
    Для реагирования на изменения параметров URL (например, ?device=thermometer).
    Для выполнения побочных эффектов, таких как фокусировка на полях ввода или автоматическое сохранение (как в случае с термометрами).
useCallback: Для мемоизации функций обратного вызова, передаваемых в дочерние компоненты (например, handleSaveEntry), чтобы предотвратить ненужные перерисовки.
useMemo: Для мемоизации вычисляемых значений (например, activeDeviceConfig, availableImportedSerials).
useRef: Для получения ссылок на DOM-элементы (например, для управления фокусом на полях ввода).

Данные передаются от родительских компонентов к дочерним через props. Обратные вызовы (callback-функции) используются для передачи данных или событий от дочерних компонентов к родительским (например, onSaveEntry из DataEntryForm в DashboardClientPage).

## 5. Ключевые функциональные блоки

### 5.1. Главная страница (src/app/page.tsx)

Отображает приветственное сообщение и описание приложения.
Содержит кнопку "Начать", по нажатию на которую открывается диалоговое окно.
Диалог выбора/добавления поверителя:
    Позволяет пользователю выбрать существующего поверителя из списка (загруженного из localStorage) или ввести фамилию нового.
    При вводе новой фамилии она автоматически обрабатывается (удаление цифр, форматирование с заглавной буквы).
    После подтверждения выбранный/добавленный поверитель сохраняется в localStorage, и пользователь перенаправляется на /dashboard.

### 5.2. Панель управления данными (src/app/dashboard/page.tsx)

Это центральная страница приложения, где происходит основная работа.

Настройки (конфигурационная карточка):
    Выбор типа устройства: Пользователь выбирает из списка устройств, определенных в src/config/devices.ts. Каждое устройство представлено карточкой с иконкой, названием и описанием. При выборе устройства карточка настроек может скрываться, и появляется форма ввода.
    Управление поверителями (InspectorManager.tsx): Встроенный компонент, позволяющий выбрать текущего поверителя из списка, добавить нового или удалить существующего. Список синхронизируется с localStorage.
Форма ввода данных (DataEntryForm.tsx):
    Адаптируется под выбранный тип устройства.
    Общая логика:
        Поле для ввода серийного номера (S/N) с возможностью "сканирования" QR-кода (ввод текста из диалога).
        Проверка на существование записи для данного S/N (и типа устройства/точки) и отображение диалога подтверждения перезаписи, если необходимо.
        Кнопки навигации ("Далее", "Назад") для многошаговых форм.
    Для стандартных устройств: Отображает набор полей (текстовые, числовые) согласно конфигурации в devices.ts.
    Для термометров:
        Пошаговый ввод: S/N -> Выбор точки поверки -> Ввод 3-х показаний для точки -> Отображение результата точки.
        Автоматическое сохранение: После отображения результата для одной точки, данные этой точки автоматически сохраняются. Если для данного S/N уже есть запись, точка добавляется/обновляется в ней. Если нет – создается новая запись. После сохранения форма сбрасывается к вводу S/N.
        Импорт S/N из Excel: Позволяет загрузить список S/N из .xlsx файла. Импортированные S/N доступны для выбора и удаляются из списка по мере завершения поверки всех трех точек для них.
        Расчеты: Автоматический расчет скорректированной температуры и вердикта ("Годен"/"Брак") на основе введенных показаний, эталонных значений и поправок из devices.ts.
    Для алкотестеров (E-200):
        Пошаговый ввод: S/N -> Выбор точки поверки -> Ввод 3-х показаний для точки -> Отображение результата точки.
        Сессионное сохранение: Данные для каждой точки добавляются в текущую "сессию" для S/N. Пользователь может поверить несколько точек. После завершения всех нужных точек нажимается кнопка "Сохранить всё", и одна запись DataEntry со всеми поверенными точками сохраняется.
        Расчеты: Автоматический расчет среднего значения и вердикта на основе введенных показаний и пределов из devices.ts.
Таблица собранных данных (DataTable.tsx):
    Отображает все сохраненные записи.
    Группировка: Записи сгруппированы по основному типу устройства (deviceType) во вкладках.
    Аккордеон: Каждая запись представлена в виде элемента аккордеона, показывающего S/N, имя устройства, поверителя и дату.
        Для термометров и алкотестеров заголовок аккордеона также отображает общий вердикт для S/N (если все точки "Годен", или есть "Брак").
    Детализация: При раскрытии аккордеона:
        Для стандартных устройств отображаются все введенные поля.
        Для термометров и алкотестеров отображается вложенная таблица с деталями по каждой поверенной точке (эталон, показания, результат точки).
    Действия:
        Удаление отдельной записи (с подтверждением).
        Очистка всех записей (с подтверждением).
        Экспорт всех данных в Excel-файл (данные по каждому типу устройства и каждой точке выгружаются на отдельные или общие листы с соответствующими заголовками).

### 5.3. Навигация и Макет

Навигационная панель (Navbar.tsx): Отображается на всех страницах, содержит логотип и ссылку на главную страницу и панель управления.
Корневой макет (layout.tsx): Определяет базовую HTML-структуру, подключает шрифты (Inter, Source Code Pro) и Toaster для уведомлений.

### 5.4. Стилизация и UI

Тема (globals.css): Определяет основные цвета приложения через CSS HSL-переменные (для светлой темы). Есть заготовка для темной темы, но она не активна.
Tailwind CSS: Используется для большинства стилей через утилитарные классы.
ShadCN UI: Предоставляет готовые, стилизованные и доступные компоненты (кнопки, инпуты, карточки, диалоги, таблицы, аккордеоны и т.д.).
Анимации: Простые анимации (например, fadeInUp, pulseSlow) определены в globals.css и tailwind.config.ts.

## 6. Важные аспекты реализации

Типобезопасность: Использование TypeScript помогает избежать многих ошибок на этапе разработки и делает код более предсказуемым.
Компонентный подход: Разделение интерфейса на мелкие, переиспользуемые компоненты улучшает структуру и упрощает поддержку.
Управление фокусом: В форме ввода (DataEntryForm.tsx) используется useRef и useEffect для автоматической установки фокуса на активное поле ввода, улучшая UX.
Обработка пользовательского ввода: Ввод для числовых полей и фамилий обрабатывается для предотвращения некорректных символов и автоматического форматирования.
Отзывчивость интерфейса: Хотя специфические детали не описаны, использование Tailwind CSS и компонентов ShadCN UI по умолчанию способствует созданию адаптивного дизайна.
Обратная связь с пользователем: Используются уведомления (toast) для информирования о результатах операций (сохранение, удаление, ошибки). Диалоговые окна используются для подтверждения критических действий. Скелетоны (Skeleton) отображаются во время начальной загрузки данных для улучшения UX.

Тестирование: Написание unit- и e2e-тестов.
